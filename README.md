## projects

<br/>

**[Track & Trust](https://github.com/krisjscott)**
&nbsp;&nbsp;`Flask` `Socket.IO` `JWT` `OCR` `SQLite` `WebSockets`
> Multi-role logistics & document platform — real-time notifications, AI-powered document risk scoring via OCR and pattern matching, role-based access control across 4 user types (government, seller, customer, driver).

<br/>

**[Govt Property Search](https://github.com/krisjscott/Hackathon-2.0)**
&nbsp;&nbsp;`TypeScript` `Express.js` `React` `Vite` `REST` `i18n`
> Full-stack property search platform built at Hackathon 2.0 — multilingual support, CSV data parsing pipeline, CORS-configured REST backend with React + Vite frontend.

<br/>

**[Ticket Booking System](https://github.com/krisjscott/Ticket-Booking)**
&nbsp;&nbsp;`Java` `Gradle` `OOP` `Multi-module` `LLD`
> CLI-based ticket booking system architected as a multi-module Gradle project — clean separation across `app`, `list`, and `utilities` modules with centralized build logic in `buildSrc`. Includes full Low Level Design.

<br/>

**[WebServer Threading](https://github.com/krisjscott/WebServer-Threading)**
&nbsp;&nbsp;`Java` `Sockets` `JMeter` `Concurrency`
> Benchmarked three server concurrency models — single-threaded, thread-per-client, and fixed thread pool — under load with Apache JMeter. Demonstrates why thread pools outperform naive threading at scale.

<br/>

**[Transformer Fine-Tuning](https://github.com/krisjscott)**
&nbsp;&nbsp;`Python` `PyTorch` `BERT` `DistilBERT` `HuggingFace`
> Fine-tuned BERT and DistilBERT on domain-specific text classification with optimized tokenization. Deployed real-time inference pipeline; evaluated with F1, precision, and recall metrics.

<br/>

**[Jarvis Autobot](https://github.com/krisjscott/jarvis_autobot)**
&nbsp;&nbsp;`Python` `OpenCV` `GPT-2` `SpeechRecognition` `Tkinter`
> Desktop AI assistant with facial authentication, voice I/O, and NLP-driven conversation via Hugging Face QA pipelines. Built as an end-to-end AI integration project.
